1. Naive Bayes

Naive Bayes is an example of gererative classifiers. Check this paper for the comparison between generative and discriminative classifiers.

1.1 Assumption & Definition

Based on a naive assumption, use Bayes' rule.

Eq.(1) also reflects the intuition behind the gererative model, where we first sample a class by P(c)P(c)P(c), then gererate words from P(d∣c)P(d|c)P(d∣c).

To simplify the computation introduced in the last expression of Eq.(1), two assumptions are made, namely the bag-of-words assumption and the naive Bayes assumption.
bag-of-words assumption: word position doesn't matter, therefore features of a document only concern identity without postion.
naive Bayes assumption: conditional probabilities P(fi∣c)P(f_i|c)P(fi​∣c) are independent, thus:

P(f1f2,...,fn∣c)=P(f1∣c)P(f2∣c)⋅...⋅P(fn∣c)P(f_1f_2,...,f_n|c)=P(f_1|c)P(f_2|c)\cdot...\cdot P(f_n|c) P(f1​f2​,...,fn​∣c)=P(f1​∣c)P(f2​∣c)⋅...⋅P(fn​∣c)

Add logarithm and walk through every word in a document:

According to Eq.(3), in Naive Bayes, predictions are made using a linear combination of input features, making Naive Bayes a linear classifier.

1.2 Training

Use MLE to compute the likelihood and prior probability:

To make inference, iterate all classes and return the one with highest probability.
Usually remove unknown words. About stop words:

In most text classification applications, however, using a stop word doesn't improve performance, and so it is more common to make use of the entire vocabulary and not use a stop word list.

1.3 Binary (Multinomial) Naive Bayes

Use binary numbers to indicate whether a specific word occurs in a document, instead of directly using the actual word count, due to the intuition that in sentiment analysis tasks, the mere existence of some word is more important that its count.

Naive Bayes with binarized features seems to work better for many text classifcation tasks.
