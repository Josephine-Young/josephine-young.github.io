1. Self-Attention Networks

Self-attention is the key innovation of transformers. At the core of an attention-based approach is the ability to compare an item of interest to a collection of other items in a way that reveals their relevance in the current context. In the case of self-attention, the set of comparisons is then used to compute an output for the current input.

3 Different roles of each input embedding during the attention process:

  • query: current focus of attention (being compared to all other preceding inputs)
  • key: preceding inputs
  • value: used to compute the output for the current focus of attention
    Intuitively, from the attention mechanism, we calculate the output at every step as follows:

score(xi,xj)=xi⋅xjαij=softmax(score(xi,xj))∀j≤iyi=∑j≤iαijxj\mathrm{score}(\mathbf{x}_i,\mathbf{x}_j)=\mathbf x_i\cdot\mathbf{x}_j\\ \alpha_{ij}=\mathrm{softmax}(\mathrm{score}(\mathbf x_i,\mathbf x_j)) \quad \forall j\le i\\ \mathbf y_i=\sum_{j\le i}\alpha{ij}\mathbf x_j score(xi​,xj​)=xi​⋅xj​αij​=softmax(score(xi​,xj​))∀j≤iyi​=j≤i∑​αijxj​

Assign different roles for input embeddings with projection matrices; Add normalization into the scoring function; Then we have:

score(xi,xj)=αij=softmax((WQxi)(WKxj)⊤dk)yi=∑j≤iαijvj\mathrm{score}(\mathbf x_i,\mathbf x_j)=\alpha_{ij}=\mathrm{softmax}\big(\frac{(\mathbf{W^Q}\mathbf x_i)(\mathbf{W^K}\mathbf x_j)^\top}{\sqrt {d_k}}\big)\\ \mathbf y_i=\sum_{j\le i}\alpha_{ij}\mathbf v_j score(xi​,xj​)=αij​=softmax(dk​​(WQxi​)(WKxj​)⊤​)yi​=j≤i∑​αij​vj​

The output yi\mathbf y_iyi​ functions as the contextualized embedding of input xi\mathbf x_ixi​.

When applying matrix multiplication for parallelization, the self-attention mechanism can be expressed as:

SelfAttention(Q,K,V)=softmax(QK⊤dk)V\mathrm{SelfAttention}(\mathbf Q, \mathbf K, \mathbf V)=\mathrm{softmax}(\frac{\mathbf Q \mathbf K^\top}{\sqrt{d_k}})\mathbf V SelfAttention(Q,K,V)=softmax(dk​​QK⊤​)V

The terminology of key, query and value, came from memory networks, a mechanism for adding an external read-write memory to networks, by using an embedding of query to match keys representing content in an associative memory.

Why replace LSTM——recurrent connections can be hard to parallelize.
Self-attention mechanism is easy to be parallelized, with each step is computed independently without involving former results. But it brings quadratic complexity at each layer when computing dot products, making it extremely expensive for long documents input. Finding more efficient attention mechanisms is an ongoing research direction.

1.2 Multihead Attention

It would be difficult for a single transfomer block to capture all of the different kinds of parallel relations among input words, Transformer further introduces multihead self-attention layers, which are essentially sets of self-attention layers residing in parallel layers at the same depth in a model, each with its own set of parameters.
Outputs from different attention heads are concatenated together followed by a linear projection WO∈Rhdv×d\mathbf{W^O}\in\mathbb{R}^{hd_v\times d}WO∈Rhdv​×d, to reduce it to the original output dimension for each token, so that multiple layers can be stacked.

1.3 Positional Embeddings

RNN-based models naturally include information about word positions since it sequentially takes in the input embedding and computes outputs. But Transformer parallelly compute the matrices and outputs at each step without invovling any relative or absolution position information, therefore it's necessary to provide extra position information. For example, when computing y3\mathbf y_3y3​, x1\mathbf x_1x1​ and x2\mathbf x_2x2​, as preceding words, both function as keys without positional distinctions.
There are some possible approaches worth considering.

  • absolute vs. relative positions
  • addition vs. concatenation between word embeddings and position embeddings
  • static function vs. learnable position embeddings
    Transformer uses a static function that combines sine and cosine functions with differing frequencies to map integer inputs to real-valued vectors.

Still, developing better position representations is an ongoing research topic.

2. Beam Search

When generating the next word conditioned on previous tokens, a naive strategy is to choose the single most probable token, like we did in n-gram models:

y^t=arg⁡max⁡w∈VP(w∣y1...yt−1)\hat y_t=\arg\max_{w\in \mathbf V}\mathbf P(w|y_1...y_{t-1}) y^​t​=argw∈Vmax​P(w∣y1​...yt−1​)

However, this way of greedy decoding might probably fall iinto local optima. To keep options open and finally find the target string with the highest probability(i.e. global optima), we need to simultaneously preserve several possible results at each step.

Dynamic Programming algorithms like Viterbi also can be used to find the optimum sequence, but not applicable to generation problems with long-distance dependencies between the output decisions.

Compared to exhaustive search (complexity of VTV^{T}VT), beam search always consumes no more than kkk decoders, with kkk as a predefined hyperparameter named “beam width”, usually between 5 and 10. If any termination symbol () was generated within the top k tokens, that branch would be removed, until all k branches reach their end or the predefined limit.

Since beam search yields sentences of differing lengths, and models usually assign lower probabilities to longer strings, it’s necessary to apply length normalization to each of the hypotheses, otherwise models will always tend to choose short strings for yyy:

score(y)=−log⁡P(y∣x)=1T∑i=1t−log⁡P(yi∣y1,...yi−1,x)\mathrm{score}(y)=-\log P(y|x)=\frac{1}{T}\sum_{i=1}^t-\log P(y_i|y_1,...y_{i-1},x) score(y)=−logP(y∣x)=T1​i=1∑t​−logP(yi​∣y1​,...yi−1​,x)
