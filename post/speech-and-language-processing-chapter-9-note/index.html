RNNs and LSTMs

2022-11-02 13 min read # NLP # Reading Note # Speech and Language Processing

Following the introduction of neural networks in Chapter 7, this chapter explains one of the most important paradigms of LLM——autoregressive encoder-decoder, in solid examples of RNN and LSTM.

Speech and Language Processing - Chapter 9 Note

Language is an inherently temporal phenomenon, possessing a temporal and sequential nature.

There are several ways for a neural model to represent time: the fixed-sized sliding window approach used by the transformer architecture, and also RNN's recurrent connections. Meanwhile, some models don't possess temporal nature and assume simultaneous access to all of the input.

1. RNN

A recurrent neural network is any network that **contains a cycle **within its network connections, meaning that the value of some unit is directly, or indirectly dependent on its own earlier outputs as an input.
A simple RNN, or Elman Networks only add a recurrent connection in the hidden layer into the feedforward network without imposing a fixed-length limit on its prior context (unlike n-gram models). The hiddern layers are recalculated for each time step while weight matrices are shared across all time steps.

hi=g(U⋅hi−1)fori∈Th_i=g(U\cdot h_{i-1}) \quad \mathrm{for} \quad i\in T hi​=g(U⋅hi−1​)fori∈T

1.1 Backpropagation through Time

Different from the backprop in FFN, the loss should not only flow through the network but also trace back to prior steps. The hidden layer at time ttt influences both the output at time ttt and the hidden layer at time t+1t+1t+1 (hence the output and loss at t+1t+1t+1).
Backpropagation through time is a general approach. During forward inference, we save the values of hidden layers at each time step, then process the sequence in reverse for loss propagation, computing and saving error terms in the hiddern layers. In accordance with this general idea, to simplify the problem, modern frameworks usually unroll the network across time steps into a feedforward computational graph. However, it demands a dynamic, scalable computational graph. If the word sequences are too long for a computational graph, we can unroll the input into manageable fixed-length segments and treat each segment as a distinct training item.

1.2 RNNs as LMs

  1. Teacher Torcing
    During training, RNN receive a input token wtw_twt​ and predict its next word y^t\hat y_ty^​t​ at each time step iteratively, however the next input token wt+1:=ytw_{t+1}:=y_twt+1​:=yt​ instead of y^t\hat y_ty^​t​. This idea that we always give the model the correct history sequence to predict rather than its own best prior prediction is called teacher forcing.

  2. Weight Tying

    Notice that for text generation tasks, the embedding layer matrix E∈R∣V∣×dhE\in \mathbb{R}^{|V|\times d_h}E∈R∣V∣×dh​ (dhd_hdh​ is the size of hidden layer) and the final layer matrix VVV maps hidden layer’s computation into a probability distribution over the vocabulary for the later softmax operation, thus V∈Rdn×∣V∣V\in \mathbb R^{d_n\times|V|}V∈Rdn​×∣V∣. These two matrices not only have an equivalent size (identical by transposition) but also function similarly. In this scenario, we could use weight tying to dispense with this redundancy and simply use matrix EEE both at the embedding layer and the softmax layer.

  3. NLP Tasks

    For tasks like** text generation** and sequence labelling, the intermidiate output at each time step yty_tyt​ is required, and there’s a softmax operation for each step; For classification tasks like sentiment analysis, the intermidiate outputs should be discarded and hidden states at each step act as embeddings. We only need one softmax operation to get the only output at the final step.

    • Autoregressive/Causal LM generation: Use an LM to incrementally generate words by repeatedly sampling the next word conditioned on its previous choices. By this definition, both RNN and n-gram belong to autoregressive models.
    • One of the key factors of good performance for an autoregressive model is providing a good context for the model to start generation. (Which leads to the creation of “prompt” mechanism in recent years ?)

2. LSTM

The way that RNN encodes history into the hidden states are relatively naive, leading to biased attention on recent inputs and obvious obliviation. As a matter of fact, the two functions simultaneously performed by hidden layers——providing information for the current output as well as carrying forward information for future decisions, should be seperated. Meanwhile, "backpropagation through time" makes the “loss flow” too long and causes vanishing gradients.

The key strategy is adding a flexible “memory mechanism” into the hidden layer instead of the hard-coded manner so that the model could learn what to "forget" or "remember". This mechanism is implemented in LSTM as a gate, each comprised of an FF layer, a sigmoid/tanh activation function and a pointwise multiplication. Adding an extra context vector as an input and output, LSTM designs several different gates:

  • forget gate: delete some information from the context that is no longer neede. (ctc_tct​ denotes the context vector)

ft=σ(Ufht−1+Wfxt)kt=ct−1⊙ft\mathbf f_t=\sigma(\mathbf U_f\mathbf h_{t-1}+\mathbf W_f\mathbf x_t)\\ \mathbf k_t=\mathbf c_{t-1}\odot\mathbf f_t ft​=σ(Uf​ht−1​+Wf​xt​)kt​=ct−1​⊙ft​

  • Compute the information to be extracted from the previous hidden states and current inputs:

gt=tanh⁡(Ught−1+Wgxt)it=σ(Uiht−1+Wixt) \mathbf g_t=\tanh(\mathbf U_g\mathbf h_{t-1}+\mathbf W_g\mathbf x_t)\\ \mathbf i_t=\sigma(\mathbf U_i\mathbf h_{t-1}+\mathbf W_i\mathbf x_t) gt​=tanh(Ug​ht−1​+Wg​xt​)it​=σ(Ui​ht−1​+Wi​xt​)

  • add gate: generate a mask to select information to be added into the current context

jt=gt⊙it \mathbf j_t=\mathbf g_t \odot \mathbf i_t jt​=gt​⊙it​

  • get new context vector:

ct=jt+kt \mathbf c_t=\mathbf j_t+\mathbf k_t ct​=jt​+kt​

  • output gate: decide what information is required for the current hidden state

ot=σ(Uoht−1+Woxt)ht=ot⊙tanh⁡(ct) \mathbf o_t=\sigma(\mathbf U_o\mathbf h_{t-1}+\mathbf W_o\mathbf x_t)\\ \mathbf h_t=\mathbf o_t\odot\tanh(\mathbf c_t) ot​=σ(Uo​ht−1​+Wo​xt​)ht​=ot​⊙tanh(ct​)

These gates are usually encapsulated into an LSTM unit in order to maintain modularity and avoid introducing further complexity into the overall architecture.

This modularity is key to the power and widespread applicability of LSTM units, which can be susstituted into other kinds of network architecture.

The hidden layers hth_tht​ can be used as input to subsequent layers in a stacked RNN or to generate an output for the final layer of the network. Within these gates, there are some intuitions behind the specific designs:

The choice of sigmoid as the activation function arises from its tendency to push its output to either 0 or 1; Combining these with a pointwise multiplication has an effect similar to that of a binary mask. When being gated, values align with values near 1 in the mask basically remain unchanged while values corresponding to lower values are essentially erased.

3. Encoder-Decoder Model with RNNs

Model architectures should suit specific tasks. Besides the former mentioned RNN architectures for NLP tasks, we can also use encoder-decoder model when input and output sequences are contextually related but of different lengths and don't align with each other in a word-to-word way, like the situation in machine translation tasks. The key idea is:

In which the context can also be expressed as the intermidiate hidden states in RNNs. Both encodes and decoders can be realized by any kind of sequence architecture.
How to turn an RNN model into encoder-decoder architectue: Input the source text with a seperator "<s>", then the target text yyy is computed by:

4. Attention Mechanism

Problem about the Vanilla Encoder-Decoder model: The context vector, namely the final hidden state usually becomes a "bottleneck": It must represent all information about the source text, but in reality it's hard to achieve, especially for long sentences. The attention mechanism is a solution to the bottleneck problem. Its innovation lies in the idea of letting the decoder to extract information from all the hidden states of the encoder instead of only the final one.
The context vector ci\mathbf c_ici​ is generated anew with each decoding step iii and takes all of the encoder hidden states into account in its derivation. Aimed at capturing relevance between each encoder state and the last decoder state, we can use dot-product attention to measure their similarity, then according to similarity scores, all encoder states get a weight factor α\alphaα (namely attention weight) to compute the context vector:

score(hi−1d,hie)=hi−1d⋅hjeαij=(softmax(hi−1d,hje)∀j∈e)ci=∑jαijhje\mathrm{score}(\mathbf h_{i-1}^d,\mathbf h_i^e)=\mathbf h_{i-1}^d\cdot h^e_j\\ \alpha_{ij}=\big(\mathrm{softmax}(h^d_{i-1},h^e_j) \quad \forall j\in e \big)\\ \mathbf c_i=\sum_j\alpha_{ij}\mathbf h_j^e score(hi−1d​,hie​)=hi−1d​⋅hje​αij​=(softmax(hi−1d​,hje​)∀j∈e)ci​=j∑​αij​hje​

The scoring function can also get pamameterized:

score(hi−1d,hie)=hi−1dWshie\mathrm {score}(\mathbf h^d_{i-1},\mathbf h^e_i)=\mathbf h^d_{i-1}\mathbf W_s\mathbf h^e_i score(hi−1d​,hie​)=hi−1d​Ws​hie​

As we've seen in Chapter 6, cosine similarity is a simple and intuitive way to measure the similarity between two vectors, namely embeddings in NLP. When exluding the impact of vector length, then we get dot-product.

By adding a weight matrix Ws\mathbf W_sWs​ in dot-product, we can train the model to focus on similarity on specific aspects between two vectors. Also, weight matrices in neural networks also help with flexibility in vector dimensions: In this way, he\mathbf h^ehe and hd\mathbf h^dhd can have different dimensions.

The current decoder hidden state is a function of last hidden state, last output and current context vector.

hid=g(y^i−1,hi−1d,ci)\mathbf h_i^d=g(\hat y_{i-1},\mathbf h_{i-1}^d,\mathbf c_i) hid​=g(y^​i−1​,hi−1d​,ci​)
